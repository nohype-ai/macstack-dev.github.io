What you might know How MacStack is different
Single-purpose tools Homebrew, your own setup.sh, mise, asdf, mackup, Stow, Dotbot, Strap MacStack allows you to manage your whole stack end to end — not just one slice of it: software installs, dotfiles, configs of AI-agents and 7 IDEs, bidirectional repo sync and more. One tool per aspect would make stack management too complex and fragile to be worth it.
File-owning tools chezmoi, nix-darwin, Ansible MacStack makes peaceful co-existence the default — not an escape hatch. It is fully non-destructive, merging into your existing files instead of owning and overwriting them — JSONC-aware, comment-preserving, down to the individual key and line. Use MacStack alongside other tools and introduce it to established or managed machines without conflict.
Parallel sandboxes Docker, Codespaces, Devbox, VMs, Tart, UTM, Parallels MacStack configures the real machines you, your team, or your customers actually work with — not some parallel isolated sandbox. Containers and VMs still have their place (MacStack happily installs them); they just solve a different problem altogether.
Enterprise tools MDM, Jamf, Workbrew, Mosyle, Kandji, Intune MacStack empowers you (not your organization) to manage your own machine. Your stack is a folder you own, containing readable files in formats you already use. No server, no enrollment, no telemetry, no per-device fee, no remote authority. Convergence runs locally when you say so — the way you defined it.

Why Use MacStack

Simple, Non-Destructive, Comprehensive

One Universal Command

mack update applies your whole stack config and updates everything

The Whole Stack

Define all aspects of the stack – far beyond software installs

Idempotent

MacStack updates only what you configure and preserves all else – down to individual entries in dotfiles and settings files.

All Configs Are Optional

MacStack requires no configs at all. Expand your stack config at your own pace.

Continue Using Homebrew

MacStack employs Homebrew so you can continue using the de facto standard dependency manager.

Lightweight

MacStack is lightweight. No docker containers, virtual machines or MDM required.

Flexible Storage

Back up your stack config wherever you want, like in a git repo, cloud folder, memory card or external drive

On Any Mac

Blends into any Mac: fresh macOS installs, established machines, managed machines, headless machines

Use What You Know

MacStack brings together the configuration systems you already use. There's almost nothing new to learn.

First-Class Comments

mack update transfers even JSON comments, so you can continue using comments as documentation.

Batteries Included

MacStack includes a few powerful customizations and helpers out of the box – no stack config required.
